# Source Boundary Manifest

This manifest distinguishes hand-authored source from generated artifacts so contributors know what belongs in version control.

## Boundary categories

### Hand-authored source assets (commit)

- Application/repository logic and tests (for example `scripts/*.py`, `tests/*.py`).
- Governance and narrative docs (for example `README.md`, `docs/*.md`, checklist files).
- Quality/runtime configuration (for example `pyproject.toml`, `.pre-commit-config.yaml`, `.github/workflows/*.yml`).

### Generated committed ledgers/assets (commit)

- `context/project_docstrings_catalog.json` when regenerated through `scripts/aggregate_project_docstrings.py`.
- `config/precommit_store/*.json` changes produced by `scripts/run_precommit_suite.py`, including `pylint_failures.json`.

### Generated local evidence caches (do not commit)

- `build/automation_contract/precommit_summary_block.txt`
- `build/automation_contract/test_summary_block.txt`
- `build/automation_contract/docstring_inventory.md`
- Other run-local outputs under `build/automation_contract/` and `reports/reasoning/pipeline/`

### Local runtime/tooling caches (do not commit)

- Virtual environments, pytest cache, type-checker caches, and temporary build products covered by root `.gitignore`.
- Editor and OS-local metadata.

## Decision rubric before staging files

1. Is the file hand-authored source? If yes, stage it.
2. Is it a generated committed ledger listed above? Stage it only when regenerated by canonical scripts.
3. Is it local evidence/cache? Keep it untracked and reference it only in summaries/PR notes.
4. If uncertain, verify via `git check-ignore -v <path>` and this manifest before committing.

## Maintenance policy

- Update this manifest whenever a new generated artifact family is introduced.
- Keep entries aligned with `docs/generated_artifact_contracts.md` and root `.gitignore` behavior.
