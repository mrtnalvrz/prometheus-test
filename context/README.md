# Context Assets

This folder stores repository-local context payloads and reusable session recipes for stateless operators.

## Contents

- `../docs/context_trigger_matrix.md`: canonical workflow-to-context load-order matrix.

- `../skills/README.md`: index of high-frequency repository skills.
- `../skills/quality-remediation/SKILL.md`: reusable remediation workflow skill.
- `../skills/checklist-audit/SKILL.md`: reusable checklist hygiene skill.
- `../skills/documentation-parity-audit/SKILL.md`: reusable documentation parity skill.
- `../skills/template-bootstrap/SKILL.md`: reusable template bootstrap skill.

- `project_docstrings_catalog.json`: machine-readable docstring export generated by `scripts/aggregate_project_docstrings.py`.
- `prompts/README.md`: index of reusable prompt assets for stateless workflows.
- `prompts/repo_audit_prompt.md`: full repository parity-audit prompt with wrapper/test closure criteria.
- `prompts/quality_remediation_prompt.md`: prompt for strict remediation loops against wrapper-surfaced failures.
- `prompts/checklist_audit_prompt.md`: prompt for actionable checklist hygiene and dependency-order enforcement.
- `prompts/template_bootstrap_prompt.md`: first-session prompt for template consumers.
- `prompts/pr_evidence_packaging_prompt.md`: prompt for preparing PR summaries with wrapper evidence blocks.
- `recipes/quality_remediation_session.md`: wrapper-first implementation/remediation workflow with explicit stop conditions.
- `recipes/checklist_audit_session.md`: checklist-governance audit workflow for dependency ordering and actionable-entry hygiene.
- `recipes/release_prep_session.md`: release-prep workflow for release-note updates, full-suite closure, and unresolved-work handoff.
- `recipes/pr_evidence_packaging_session.md`: PR evidence packaging workflow for summary-block capture and unresolved-work escalation.
- `recipes/documentation_parity_audit_session.md`: repeatable documentation parity audit workflow with command/path and implementation checks.

## Recipe usage contract

1. Start with `AGENTS.md` and `docs/agent_bootstrap/operator_context_injection.md`.
2. Pick the recipe that matches the session objective.
3. Execute commands exactly as documented (wrapper-first, no direct hook shortcuts).
4. If work cannot close in-session, add granular follow-up entries in `Final-Productization-Checklist.md`.

## Encoding and timestamp hygiene

- Save all context assets as UTF-8.
- Derive any datestamps/timestamps from Git metadata or trusted system tooling rather than model-relative time.


## Prompt library contract

Every prompt under `context/prompts/` must include purpose, ingestion order, canonical commands, evidence requirements, and closure criteria so operators do not reconstruct workflow instructions from memory.
